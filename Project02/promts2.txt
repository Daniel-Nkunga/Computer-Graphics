// Returns true if two circles overlap.
// Collision occurs when distance between centers < sum of radii.
bool checkCollision(tuple<float,float,float> posA, float rA,
                    tuple<float,float,float> posB, float rB)
{
    float dx = std::get<0>(posA) - std::get<0>(posB);
    float dy = std::get<1>(posA) - std::get<1>(posB);
    float dz = std::get<2>(posA) - std::get<2>(posB);
    float distance = std::sqrt(dx*dx + dy*dy + dz*dz);
    return distance < (rA + rB);
}

The rotation needs to happen in pacman.cpp. The idea is to track the last direction pacman moved and use glRotatef to rotate the model before drawing. Since the mouth already faces right (0°), we just need to calculate the angle from dx/dy using atan2.

Only two files changed. Here's what was added:

pacman.hpp — a float facingAngle = 0.0f private member to store the current facing direction in degrees (0° = right, 90° = up, 180° = left, 270° = down).

pacman.cpp — two changes:

In update(), whenever pacman actually moves, atan2(dy, dx) computes the angle from the movement vector and stores it in facingAngle. This means diagonal movement is handled correctly too — moving up-right faces 45°, up-left faces 135°, etc.
In display(), a glPushMatrix/glPopMatrix block applies glRotatef(facingAngle, 0, 0, 1) around pacman's center before drawing. The translate-rotate-translate pattern is necessary because glRotatef rotates around the origin, not around pacman's position.
