Your intuition is right but the loop has a couple of issues — h (the base noise) should be re-sampled at each frequency rather than just scaled, and amplitude should increase while frequency decreases (it's currently the opposite). This is called fBm (Fractional Brownian Motion) — you're summing the same noise field sampled at increasing frequencies with decreasing amplitudes... but yours has it inverted.

Here's the corrected height calculation:

cpp
float nx = (float)x / (SIZE - 1) * 2.0f - 1.0f;
float ny = (float)y / (SIZE - 1) * 2.0f - 1.0f;

// fBm: sum octaves of noise, each at a higher frequency but lower amplitude
float nz   = 0.0f;
float a    = 1.0f;   // amplitude  — starts high, decreases
float f    = 1.0f;   // frequency  — starts low,  increases
float norm = 0.0f;   // track total amplitude for normalisation

for (int i = 0; i < 8; i++)
{
    // Sample the noise grid at scaled coordinates for this octave
    int sx = min((int)(x * f) % SIZE, SIZE - 1);
    int sy = min((int)(y * f) % SIZE, SIZE - 1);
    nz   += a * (float)perlin_grid[sy][sx];
    norm += a;

    a *= 0.5f;   // amplitude halves  each octave (persistence)
    f *= 2.0f;   // frequency doubles each octave (lacunarity)
}

nz /= norm; // normalise back to [0, 1]
coordinates[y][x] = { nx, ny, nz };
The key corrections vs your version:

a decreases (*= 0.5) and f increases (*= 2.0) — your original had it backwards (a -= 0.1, f += 0.1 which also goes negative/too-large quickly)
Re-sampling the noise at (x*f, y*f) rather than just multiplying h by f — without re-sampling you're not actually adding detail, just scaling the same value
Normalising by total amplitude so the result stays in [0, 1] regardless of octave count
That said, since your PerlinNoise::generate() already does fBm internally with octaves/persistence/lacunarity parameters, you could also just tune those instead of doing it manually in main. The manual approach here gives you per-vertex control if you want to add things like domain warping or erosion later.
